## 听众可能关心的三个问题（心里先有答案）

1. **和 ChatGPT 上传文件有什么区别？**  
   我们不是单轮聊天，而是：**多文件会话绑定、专用模式（理解/编辑/填表）、工作流批处理、结果文件落地（另存为）**，面向办公流水线而不是闲聊。

2. **数据会不会上传到云端？**  
   桌面版默认数据在本地（`%APPDATA%\document-intelligence-desktop-electron\data`）；**只有调用大模型时**会把文档片段发给所配置的 API 供应商（DeepSeek / 智谱等），Key 由用户在设置页自行填写。

3. **能处理哪些格式？**  
   文档库支持 **PDF / Word / Excel / Markdown / 文本**；对话模式对格式有不同要求（例如文档编辑需 docx）；填表典型组合是 **Excel 数据源 + Word/Excel 模板**。

## 第 3 页 · 产品全景（三大入口）

**屏上结构（建议三列卡片）**

| 模块 | 用户做什么 | 系统做什么 |
|------|------------|------------|
| **文档库** | 上传、分空间管理资料 | 统一入口，供对话/工作流选用 |

**口述（约 40 秒）**

> 产品上有三个主界面。文档库负责「资料从哪来」；智能对话负责「我现在要干什么」——默认聊天、围绕文档问答、按自然语言改 Word、或者做实体提取和表格填表；工作流负责「把重复任务固化」——比如固定成「选文档 → AI 翻译 → 导出 PDF」。桌面版还特别做了系统级「另存为」，生成文件不强制进文档库，用户自己选目录。

## 第 4 页 · 核心能力 × 智能体分工

**屏上表格（精简版）**

| 智能体 | 能力 | 典型文档 |
|--------|------|----------|
| **A** | 文档理解、按自然语言编辑 | Word、文本、Markdown、表格 |
| **B** | 实体/字段抽取，输出结构化数据 | Word、PDF、文本、Markdown |
| **D** | 从 Excel 筛选数据并填入模板 | Excel 数据源 + Word/Excel 模板 |

**口述（约 30 秒）**

> 系统按任务类型调度不同智能体：理解、改文档走 A；把非结构化内容抽成结构化数据走 B；表格填表走 D。桌面版**全部落在本机**——会话、文档库、工作流配置和生成文件都在本地目录，**不需要数据库**，装好就能演示。

**可展开素材**

- **混合模式**：一次会话里可组合「理解 + 填表」，适合「先读懂再填入模板」。  
- **模板优先填表**：先识别模板要填哪些列，再到数据源里找对应字段。  
- **文档编辑**：完成后在界面中「另存为」到用户自选路径。

**口述（约 30 秒）**

# 常见问题（Q&A 预备，不必全讲）

**Q：没有网能用吗？**  
A：界面和文档库可离线；理解/翻译/填表依赖所配置 LLM API，需网络。未来可接本地模型。

**Q：支持 Mac 吗？**  
A：当前交付以 Windows 安装包为主；桌面壳基于 Electron，架构上可扩展其他平台。

**Q：和工作流「翻译」结果仍是中文？**  
A：检查工作流里 AI 翻译节点是否选了目标语言「英语」；演示前建议先跑通一遍样例。

**Q：一定要联网吗？**  
A：界面与本地文档库可离线使用；调用大模型翻译/理解时需要网络（或未来本地模型）。

# 附录：架构与数据流（备查）

**本地数据存什么（概念层）**

| 内容 | 说明 |
|------|------|
| 模型与 API 配置 | 用户填写的 Key、模型名等 |
| 文档库 | 上传文件的索引与空间划分 |
| 会话与消息 | 对话历史、已选文件 |
| 工作流 | 节点编排与执行记录 |
| 生成物 | 用户「另存为」后的文件在用户自选目录 |

**一次对话的大致数据流**

文档库选文件 → 前端选模式（理解/编辑/填表）→ 本地服务解析文档 → 调度对应智能体 → 调用大模型 → 流式返回前端 → 若有文件产出则「另存为」

**一次工作流的大致数据流**

输入节点选库内文件 → 中间 AI 节点处理（翻译/摘要等）→ 输出节点生成 PDF/Markdown 等 → 可写入文档库或由用户另存
